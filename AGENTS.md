# Session change summaries for AI agents

After code or config changes, agents append short notes here. When this file grows past ~20 lines, **replace** the changelog with an updated condensed section instead of keeping one bullet per file forever.

## Agent Workflow Rules

- **ALWAYS use the `question` tool** when clarification is needed, when weighing tradeoffs, or when the user might have a preference. Do not ask questions inline in text responses—use the dedicated tool.

- **NEVER** hardcode the domain name in the code.

## Condensed changelog

- **Config & layout**: `.env` holds secrets (e.g. `OPENROUTER_API_KEY`); shared defaults live in `src/core/constants.py`. Packages: `src/core`, `src/media`, `src/llm`, `src/ffmpeg`, `src/startup`; orchestration in `src/app/pipeline.py`. Legacy top-level shim modules were removed.
- **Transcription, snippet, title & overlay packages** (`packages/`): `packages/openrouter_transport/` — shared transport for chat requests (was `src/llm/client.py`). `packages/sr_transcription/` — `transcribe_and_save` API, prompt, format validation; imports `openrouter_transport` and `src.core.constants` for `AUDIO_FORMATS` and `OPENROUTER_DEFAULT_MODEL`. `packages/sr_snippet/` — `create_silence_removed_snippet` / `create_silence_removed_audio`; uses `build_trim_plan` from `packages/sr_trim_plan/`, FFmpeg builders, and `src.ffmpeg.silence_removed_runner`. `packages/sr_title/` — title generation APIs + prompts; imports `openrouter_transport` and `OPENROUTER_DEFAULT_MODEL`. `packages/sr_title_overlay/` — `build_title_overlay` only (Pillow + Google Fonts + Arabic shaping); layout tunables in package `constants.py`. `packages/sr_trim_plan/` — shared `TrimPlan` + `build_trim_plan` policy API for snippet/final trim. `packages/sr_telegram_notify/` — optional `notify_final_encoding_started` / `notify_final_output_ready` text notifications via Bot `sendMessage` (env `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID`; optional `TELEGRAM_API_BASE`). `src/llm/__init__.py` re-exports `sr_transcription` and `sr_title` only (no `audio_for_llm`). `pyproject.toml` Hatch `packages` includes `src`, `packages/sr_transcription`, `packages/sr_snippet`, `packages/sr_title`, `packages/sr_title_overlay`, `packages/sr_trim_plan`, `packages/openrouter_transport`, `packages/sr_telegram_notify`.
- **Planning doc**: Historical encapsulation notes referred to `openrouter_transport` + `sr_transcription` (+ title in `src/llm`); transcription and title LLM modules now live under `packages/` with stable import names.
- **FFmpeg layer**: Central builders, runners (including progress parsing), probing, silencedetect helpers, filter-graph utilities, and transcode command assembly; `src/ffmpeg/silence_removed_runner.py` runs filter-script silence-removed encodes for audio snippet + video trim. **`run_with_progress`**: stdout piped for `-progress` lines; **stderr inherited** so FFmpeg errors print to the terminal (no stderr capture/thread). **`format_ffmpeg_process_failure`**: short `RuntimeError` text pointing at stderr. Scripted graphs use `-/filter_complex`; legacy `-filter_complex_script` fallback was dropped (FFmpeg 8+ assumed). Final video HEVC prefers **`hevc_qsv`** when listed and the startup probe succeeds; uses **`libx265`** only when **`hevc_qsv`** is not listed by FFmpeg (see **Encoding**). **Silencedetect**: chained primary+edge in one decode when stderr exposes two filter labels (`detect_primary_and_edge_silence_points`); fallback to two passes; **skip** silencedetect when there is no audio stream; silence parse uses `run(..., capture_output=True)` (fixes subprocess usage on audio-only paths). **Video-only inputs**: lavfi-backed concat graphs, silent-audio generation, and related transcode helpers. Removed unused fixed-window extract helpers (`build_audio_window_extract_command`, `build_first_5min_audio_*`).
- **Encoding**: `src/ffmpeg/encoding_resolver.py` prefers **`hevc_qsv`** in ICQ mode (`-global_quality 20`, `-preset slow`); falls back to **`libx265`** (`-crf 24`, `-preset slow`, `hvc1` + faststart) only when **`hevc_qsv`** is not listed. If **`hevc_qsv`** is listed but the startup probe encode fails, startup fails. Probe uses `yuv420p` lavfi; startup also fails if no usable encoder can be resolved (e.g. missing fallback or failed fallback probe).
- **Silence & trim**: `TrimPlan` / `build_trim_plan` unify target vs non-target policies. Edge handling still uses one cached edge scan plus shared normalization across snippet, target, and non-target paths. Live target mode now uses a fixed two-stage binary search: thresholds `-60.0..-35.0 dB` on a `0.1 dB` grid at `min_silence_len=0.100s`, then padding search from `0.060s` in `0.01s` increments; if target remains unreachable or the final fallback probe is still invalid, the planner returns best effort at `-35.0 dB` with `0.060s` padding, preserving the full file if needed, and never truncates. Because segment building still keeps padding on both sides of a cut, the practical removable-silence floor stays around `0.120s`.
- **Transcription snippet**: Phase 1 uses `create_silence_removed_snippet` from `sr_snippet` with fixed snippet constants (ignores `--non-target-noise-threshold` / `--non-target-min-duration`); max length `SNIPPET_MAX_DURATION_SEC` (180s). Snippet path reuses the same edge policy as final trim.
- **Transcription input**: `transcribe_media` accepts audio paths only (`AUDIO_EXTENSIONS`); snippet `.ogg` from Phase 1 is passed directly to `transcribe_and_save` (no video→audio helper).
- **Pipeline run**: Always executes phases 1–3 (transcription, title, final MP4); video encoder profile is resolved at startup via `resolve_video_encoder()` (QSV-primary; see **Encoding**). Phase 3 optional Telegram text at encode start (`notify_final_encoding_started`) and after success (`notify_final_output_ready`) (see **Transcription… packages**). The former `--llm-only` mode, `temp/titles.txt` run log, and `pwsh/Start-VerticalLlmDryRun.ps1` were removed.
- **Telegram notifications**: `packages/sr_telegram_notify` uses single-word status prefixes (`STARTED`, `READY`, `AUDIO`, `VIDEO`) instead of emoji markers, with the status/counter on line one and `basename: title` on line two so pipeline notifications stay consistent and plain-text friendly.
- **Transcript gating**: `transcribe_and_save` does not write a file when the model returns empty/whitespace-only text (`RuntimeError`). `generate_title_from_transcript` / `generate_title_with_openrouter` raise `RuntimeError` on empty/whitespace-only transcript (no title file written). `is_transcript_done` requires a transcript file with non-whitespace UTF-8 content (invalid UTF-8 is treated as not done). Phases 2–3 require a usable transcript before title/output.
- **LLM client**: OpenRouter requests default to capping input/context and output size (10k tokens each), with compatibility handling if the API rejects some fields.
- **Default models**: Transcription and title flows default to `OPENROUTER_DEFAULT_MODEL` in `src/core/constants.py` (`google/gemini-3.1-flash-lite-preview`) unless callers override.
- **Title generation**: Prompts require verbatim, beginning-only title spans from the full transcript. One model call emits a small JSON array of candidates; one call returns per-candidate `verbatim_score` and `correctness_score`; highest combined score wins with deterministic tie-breaks. **No** separate honorific add/check LLM step after selection.
- **Phase 3 title overlay**: Pipeline passes `title_path` / `title_font` into trim; forces encode (no stream copy) when overlay is used. **PNG overlay** (`packages/sr_title_overlay/`): Google Fonts CSS2 + TTF cache; Pillow render; Arabic via `arabic-reshaper` + `python-bidi`; filter graph uses PNG concat with `shortest=1`. **Layout**: sixths-based band (e.g. H/6–H/3) from `src.core.constants`; multi-line layout tunables in `sr_title_overlay.constants`. **Optional logo**: repo `logo/logo.png`, `colorchannelmixer` alpha, uniform `scale` to `video_w * LOGO_OVERLAY_WIDTH_FRACTION_OF_VIDEO` vs intrinsic width (default full width), top-aligned, default zero margin and full alpha gain; composited **under** the title strip when both are used; `probe_video_dimensions` plus `probe_ffmpeg_can_decode_image_frame` (one FFmpeg-decoded frame) must pass or logo is skipped with a warning. **Metadata**: final MP4 `comment` = source filename for editor-driven cleanup; legacy `SILENCE_REMOVER_SOURCE` still matched. **CLI**: `--title-font`.
- **Title editor removal**: The local FastAPI title editor (`src/title_editor/`, `serve_titles.py`, `pwsh/Start-VerticalTitleEditor.ps1`, `--title-editor` CLI flag) has been removed. Title editing is now handled by the VPS-based MP3 Manager system in the `remote/` directory.
- **Documentation**: `README.md` and `ALGO.md` cover architecture, CLI, snippet/edges, encoding, title rules, overlay geometry, and video-only behavior.
- **Media Manager Backend** (`remote/`): FastAPI service with project routes under `/projects/{token}/{project}/` and admin dashboard at `/admin/{admin_token}/`. Endpoints: GET/POST/PUT/DELETE `/projects/{token}/{project}/api/files`, GET `/projects/{token}/{project}/stream/{id}`, GET `/admin/{admin_token}/api/projects`. SQLite database with JSON tags array, flat file storage in `storage/audio/` and `storage/video/`. Tag-based virtual organization with auto-trash for empty tags. Supports both audio (fixed tags: todo/ready/all/trash) and video (freeform tags: FB/TT/custom). Admin dashboard shows aggregated project stats with direct links to projects.
- **Media Manager Pipeline Integration** (`packages/sr_media_manager/`): Renamed from `sr_mp3_manager`. New 5-phase workflow: Phase 3 uploads audio with `tags: ["todo"]`, Phase 4 creates video locally, Phase 5 uploads video with `tags: ["FB", "TT"]` only when audio is marked `ready`. Two-way sync at startup fetches edited titles from Media Manager and triggers re-encode if changed. `pyproject.toml` updated to include `sr_media_manager` package.
- **Pipeline 5-Phase Workflow**: Phase 1 (transcription), Phase 2 (title generation), Phase 3 (audio upload for review), Phase 4 (video creation local only), Phase 5 (video delivery when approved). `src/app/pipeline.py` updated with `run_audio_upload_phase`, `run_video_upload_phase`, and `sync_titles_from_api` for two-way title sync.
- `src/media/trim.py`: Modified `trim_single_video()` to accept `encoder: str` parameter (default `"libx265"`) instead of `VideoEncoderProfile | None`. Added import for `get_encoder_config`. Updated encoder resolution logic to use `.codec` attribute when falling back to `resolve_video_encoder()`. Updated all `command_label` references to use the string encoder directly.
